## Purpose

How release-facing status is recorded. Hey Ralli does not maintain a separate marketing changelog file yet; **shipped truth lives in the Feature list**.

## What to update on each release / feature PR

1. **[feature-list.md](./feature-list.md)** — move items between shipped / partial / deferred; date the doc header.
2. Related living doc per [GOVERNANCE.md](../GOVERNANCE.md) (integration, eng, ops, or QA).
3. Optional: note user-visible changes in the GitHub PR / release description (source for future public notes).

## Future

A dated `CHANGELOG.md` or in-app “What’s new” can be added later; until then, Feature list + git history are the record.
